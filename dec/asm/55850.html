55850: Cycle Border from Screen
Drives the per-frame border colour cycling effect. Reads successive pixel bytes from the screen buffer pointer at 50467 ANDs each byte with 24 to extract the paper colour bits, and writes the result to the border port 254. Between each byte a variable delay is run, its period advanced by the step value at 50475. After the outer loop the frame counter at 50471 is decremented; when it reaches the threshold at 50477 Entity_State_Flag is set to 1 (step complete). When the frame counter reaches zero, Entity_State_Flag is set to 2 (all complete). The alternate entry at Cycle_Border_Toggle_Loop drives the same counter machinery without sampling screen data, instead XOR-toggling the border between two colours.
